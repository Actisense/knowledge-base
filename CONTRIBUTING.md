## 🤝 Code of Conduct

Please be respectful and constructive in all interactions. We want this to be a welcoming space for developers of all experience levels.

---

## 💡 What Can I Contribute?

We welcome contributions of all kinds, including:

- **Product knowledge** — tips, quirks, and useful configurations for Actisense hardware
- **Integration guides** — step-by-step walkthroughs for connecting Actisense devices to third-party systems
- **Examples & recipes** — creative or practical ways to use Actisense products
- **Troubleshooting entries** — fixes for common issues you've encountered
- **FAQ additions** — questions and answers that would help other developers
- **Corrections & improvements** — fixing typos, outdated information, or unclear explanations

## 🚀 Getting Started

1. **Fork** this repository to your own GitHub account.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/<your-username>/knowledge-base.git
   ```
3. **Create a branch** for your contribution:
   ```bash
   git checkout -b my-contribution
   ```
4. Make your changes, then **commit** them:
   ```bash
   git commit -m "Add guide for integrating NGT-1 with OpenCPN"
   ```
5. **Push** to your fork:
   ```bash
   git push origin my-contribution
   ```
6. Open a **Pull Request** against the `main` branch of this repository.

---

## 📝 Submitting a Contribution

When opening a Pull Request, please:

- Give it a clear, descriptive title
- Briefly describe what you've added or changed and why
- Link to any relevant Actisense product pages, SDK docs, or issues

For larger contributions (e.g. a new section), consider opening an **Issue** first to discuss the approach before writing everything up.

---

## ✍️ Style Guide

- Write in clear, plain English — assume the reader is a competent developer but may be new to Actisense products
- Use Markdown headings (`##`, `###`) to structure documents
- Use fenced code blocks with a language identifier for all code snippets
- Keep filenames lowercase with hyphens, e.g. `ngt-1-opencpn-guide.md`
- Place files in the most relevant directory (`products/`, `integration-guides/`, `examples/`, `troubleshooting/`)
